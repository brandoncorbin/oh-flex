Oh-Flex, a javascript-less, CSS toolkit was created to give non-frontend developers tools to avoid touching CSS.

What's the Point?

I work with a lot of large companies who are trying to get their hands around an overaching UI process. Most companies have a bunch of developers who, honestly, hate working on the front end. They've resorted to throwing Bootstrap or some other UI library and call it done.

As the developers start creating more and more functionality, the find themselves writing a bunch of hacky CSS just to get the problem solved as fast as possible. Each time a developer implements a CSS hack, an angel loses its wings. Oh-Flex is a set of classes to solve this problem.

This is a set of mixins and classes I found to use over and over in my projects. This page is an example of what you can do with these classes. Not a good example tho! Here's why... This page is completely designed with just class names. No external styles besides Oh Flex...

What's included?
Insane Flexibility

Oh-Flex should work with any existing UI library - including Bootstrap, Angular Material, SemanticUI, etc. Everyhting is prefixed as to not mess up any existing design.

Flexbox

.f-rubber, .f-flex, .f-row, .f-column, .f-grow, .f-shrink

List Item

A sensible and simple list item using flexbox

Toolbar

A highlevel toolbar class for creating flexible and responsive toolbars.

Layout Control

Have sticky headers and footers, side menus and the likes - without a bunch of crazy code.

Margin and Padding

Pixel level control of padding and margin with shortcuts. .f-p-20px adds 20px padding on all sides. While .f-p-b-10px adds padding to the bottom.

Font Size and Colors

Pixel level font control .f-text-20px .f-text-bolder .f-text-bold
